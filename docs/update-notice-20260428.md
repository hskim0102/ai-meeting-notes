# NoteFlow 업데이트 공지 (2026-04-28)

안녕하세요. NoteFlow 업데이트 내용을 안내드립니다.

---

## 주요 변경사항

### 1. 회의록 삭제 기능 추가

회의 상세 화면에서 회의록을 직접 삭제할 수 있습니다.

- 편집 버튼 옆 **삭제** 버튼 클릭
- 확인 모달에서 삭제 의사 재확인 후 진행
- 삭제 시 Dify 지식 베이스의 연결된 RAG 도큐먼트도 함께 삭제됩니다
- RAG 삭제에 실패한 경우 회의록은 삭제되지 않으며 오류 메시지가 표시됩니다

### 2. Dify RAG 연동 완성 (생성 · 수정 · 삭제)

회의록과 Dify 지식 베이스 간 데이터 동기화가 전체 생명주기에 걸쳐 자동 처리됩니다.

| 동작 | Dify 처리 |
|------|-----------|
| 회의록 **저장** | Dify 지식 도큐먼트 **생성** |
| 회의록 **수정** | Dify 지식 도큐먼트 **업데이트** |
| 회의록 **삭제** | Dify 지식 도큐먼트 **삭제** |

### 3. Dify 에이전트 구분값(gubun) 변경

Dify 워크플로우에 전달하는 `gubun` 값이 변경되었습니다.

| 이전 | 이후 | 의미 |
|------|------|------|
| `'1'` | `'C'` | Create (생성) |
| `'2'` | `'U'` | Update (수정) |
| _(없음)_ | `'D'` | Delete (삭제) |

> **Dify 워크플로우 담당자 확인 필요:** 에이전트 쪽에서 `gubun` 값 `'C'` / `'U'` / `'D'` 분기 처리가 구현되어야 합니다.

### 4. Dify 입력 필드명 변경

Dify 워크플로우 입력 폼 변수명에 맞게 회의 데이터 필드명이 변경되었습니다.

| 이전 | 이후 |
|------|------|
| `meetings_note` | `test` |

### 5. [임시] RAG 수동 생성 버튼 동작 개선

기존 회의록 `document_id` 채우기용 임시 버튼의 gubun 판단 기준이 변경되었습니다.

- **이전**: `document_id` 유무로 C/U 판단
- **이후**: `meeting_rag_docs` row 존재 여부로 C/U 판단 (row 없으면 C, row 있으면 U)
