## 1. 프로젝트 개요

### 1.1 배경 및 필요성

사내 임직원들은 일 평균 2~3회의 회의에 참석하며, 회의 후 수기로 회의록을 작성하고 액션 아이템을 추적하는 데 상당한 시간을 소모하고 있다. 기존 회의록 작성 프로세스는 다음과 같은 문제를 안고 있다:

- **생산성 저하**: 회의 내용 정리에 회의 시간의 30~50% 추가 소요
- **정보 누락**: 수기 작성 시 핵심 논의사항 및 결정사항 누락 빈번
- **추적 불가**: 액션 아이템의 체계적 관리 부재로 후속 조치 지연
- **검색 한계**: 과거 회의 내용의 검색 및 재활용이 사실상 불가능

### 1.2 프로젝트 목표

AI 기반의 스마트 회의록 Agent를 도입하여 **회의 녹음 → 텍스트 변환 → AI 요약 → 액션 아이템 자동 추출**의 End-to-End 자동화 파이프라인을 구축한다.

| 목표 | 기대 효과 |
|------|-----------|
| 회의록 작성 자동화 | 회의록 작성 시간 80% 이상 절감 |
| 액션 아이템 자동 추출 | 후속 조치 누락률 감소, 업무 추적성 확보 |
| 회의 검색 및 아카이빙 | 과거 회의 내용 즉시 접근 및 활용 가능 |
| 화자 구분 및 발언 추적 | 발언 책임 명확화, 맥락 이해도 향상 |

### 1.3 대상 사용자

- **1차 사용자**: 사내 전 임직원 (회의 참석자)
- **2차 사용자**: 팀 리더 / PM (액션 아이템 관리 및 추적)
- **관리자**: IT 운영팀 (시스템 관리 및 모니터링)

### 1.4 제품 범위

| 포함 | 미포함 (향후 고려) |
|------|---------------------|
| 실시간 마이크 녹음 | 화상회의 플랫폼 직접 연동 (Zoom, Teams) |
| 오디오 파일 업로드 | 다국어 동시통역 |
| STT 텍스트 변환 | 사내 메신저 자동 알림 |
| AI 요약 및 액션 아이템 추출 | 외부 캘린더 동기화 |

## 2. 기술 스택

### 2.1 프론트엔드

| 기술 | 버전 | 용도 |
|------|------|------|
| **Vue.js 3** | ^3.5 | SPA 프레임워크 (Composition API, `<script setup>`) |
| **Vue Router** | ^4.6 | 클라이언트 사이드 라우팅 |
| **Tailwind CSS** | v4 | 유틸리티 기반 스타일링 |
| **Vite** | ^8.0 | 빌드 도구 및 개발 서버 |

### 2.2 백엔드

| 기술 | 버전 | 용도 |
|------|------|------|
| **Node.js** | 20 LTS+ | 런타임 환경 |
| **Express.js** | ^5.2 | REST API 서버 |
| **Multer** | ^2.1 | 멀티파트 파일 업로드 처리 |
| **fluent-ffmpeg** | ^2.1 | 오디오 파일 분할 (Chunking) |
| **mysql2** | ^3.20 | MySQL 데이터베이스 드라이버 (Promise 기반) |

### 2.3 AI 연동

| 기술 | 용도 |
|------|------|
| **OpenAI Whisper API** | 음성-텍스트 변환 (STT) |
| **Dify Workflow API** | 회의록 요약, 액션 아이템 추출, 키워드 JSON 생성 |

### 2.4 인프라 (현재 → 향후)

| 구분 | 현재 (프로토타입) | 향후 (운영) |
|------|-------------------|-------------|
| 호스팅 | Localhost (dev) | 프라이빗 클라우드 / 사내망 |
| 스토리지 | 로컬 파일시스템 | 오브젝트 스토리지 (S3 호환) |
| DB | MySQL 8.x (`mysql2/promise`, `localhost:30306`) | PostgreSQL 또는 클러스터 MySQL |
| 인증 | 없음 | SSO / LDAP 연동 |

### 2.5 모듈 구성 (ESM)

프로젝트 전체는 `"type": "module"` 기반의 ESM(ECMAScript Modules)으로 구성되며, CommonJS 모듈은 사용하지 않는다.

| **설명** | 사용자가 브라우저에서 직접 마이크를 통해 회의 내용을 실시간 녹음한다 |

- 브라우저 마이크 권한 요청 및 권한 거부 시 안내 메시지 표시

| **기능 ID** | F-UPL-001 |
| **기능명** | 드래그 앤 드롭 오디오 파일 업로드 |
| **설명** | 기존 녹음 파일(MP3, WAV, M4A, WebM 등)을 업로드하여 텍스트 변환을 수행한다 |
| **기술 구현** | Multer (서버), Drag & Drop API (클라이언트) |

### 4.3 대용량 파일 Chunking 및 STT 변환

| 항목 | 상세 |
|------|------|
| **기능 ID** | F-STT-001 |
| **기능명** | 오디오 분할 및 STT 텍스트 변환 |
| **설명** | 25MB 이상의 대용량 오디오 파일을 자동 분할하고 Whisper API를 통해 텍스트로 변환한다 |
| **기술 구현** | fluent-ffmpeg (분할), OpenAI Whisper API (STT) |
| **서비스** | `audioSplitter.js`, `whisperService.js` |

**상세 요구사항**:

- **분할 기준**: 25MB 초과 시 20MB 단위로 자동 분할
- **분할 방식**: ffmpeg를 이용한 시간 기반 분할 (오디오 품질 유지)
- **병렬 처리**: 분할된 청크를 순차/병렬로 Whisper API에 전송
- **텍스트 병합**: 각 청크의 STT 결과를 시간순으로 정렬 및 병합
- **임시 파일 관리**: 업로드 파일 및 분할 청크는 처리 완료 후 자동 삭제 (`uploads/`, `temp/`)
- **에러 처리**: 개별 청크 실패 시 재시도 로직 (최대 2회)
- **타임스탬프 보존**: 각 청크별 시작 시간 오프셋을 반영하여 전체 타임라인 유지

**처리 흐름**:

```
오디오 파일 (원본)
    │
    ├─ ≤ 25MB → Whisper API 직접 호출
    │
    └─ > 25MB → ffmpeg 분할 (20MB chunks)
                    │
                    ├─ chunk_001.webm → Whisper API → 텍스트 1
                    ├─ chunk_002.webm → Whisper API → 텍스트 2
                    └─ chunk_00N.webm → Whisper API → 텍스트 N
                                                        │
                                                   [시간순 병합]
                                                        │
                                                   완성된 Transcript
```

### 4.4 Dify API 연동 — AI 구조화 요약

| 항목 | 상세 |
|------|------|
| **기능 ID** | F-SUM-001 |
| **기능명** | AI 기반 회의록 구조화 요약 |
| **설명** | STT 결과 텍스트를 Dify Workflow API에 전달하여 구조화된 회의록 결과를 생성한다 |
| **기술 구현** | Dify Workflow API (HTTP POST) |
| **서비스** | `difyService.js` |

**Dify 출력 구조**:

```json
{
  "summary": "회의 핵심 내용 요약 (TL;DR)",
  "action_items": [
    {
      "id": "uuid",
      "task": "액션 아이템 내용",
      "assignee": "담당자",
      "due_date": "마감일 (추정)",
      "priority": "high | medium | low",
      "status": "pending"
    }
  ],
  "keywords": ["키워드1", "키워드2", "키워드3"],
  "key_decisions": [
    {
      "decision": "결정 사항",
      "context": "결정 배경"
    }
  ],
  "participants": ["참석자1", "참석자2"]
}
```

**상세 요구사항**:

- Dify API Key는 서버 사이드에서만 관리 (클라이언트 노출 금지)
- API 타임아웃: 120초 (LLM 처리 지연 대비)
- 응답 파서: JSON-string 형태와 개별 필드 형태 모두 처리 가능
- 요약 실패 시 원본 텍스트 보존 및 재시도 옵션 제공

#### 4.7.2 상세 요구사항

**회의실 관리**:

- **회의실 정보 표시**: 이름, 위치(건물/층), 수용 인원, 보유 장비 (프로젝터, 화이트보드, 화상회의 장비, 마이크 등)
- **회의실 상태**: 사용 가능 / 사용 중 / 점검 중 실시간 상태 표시
- **회의실 이미지**: 회의실 사진 등록 (선택사항)

**예약 생성**:

- 날짜, 시작 시간, 종료 시간, 회의실, 회의 주제, 참석자 입력
- 최소 예약 단위: 30분, 최대 예약 시간: 4시간
- **반복 예약**: 매주 / 격주 / 매월 (지정 요일) 반복 생성 지원
- 예약 시 회의실 가용 여부 실시간 검증

**예약 수정/취소**:

- 본인이 생성한 예약에 한해 수정 및 취소 가능
- 과거 시점의 예약은 수정/취소 불가 (읽기 전용)
- 반복 예약 수정 시: "이 일정만" / "이후 모든 일정" 선택 옵션

**충돌 방지**:

- 동일 회의실 + 동일 시간대 중복 예약 차단 (서버 사이드 검증)
- 예약 폼에서 실시간 충돌 여부 표시 (클라이언트 사이드 프리체크)
- 동시 예약 요청 시 선착순 처리 (Optimistic Locking)

**회의록 연동**:

- 예약된 회의 시간에 생성된 회의록과 자동 매칭 (시간대 기반)
- 수동 연결: 기존 회의록을 예약 건에 직접 연결 가능
- 회의 상세 화면에서 연결된 예약 정보 표시

**알림**:

- 회의 시작 10분/30분 전 브라우저 Push 알림 (Notification API)
- 예약 확인/변경/취소 시 참석자 알림

#### 4.9.2 상세 요구사항

**빠른 검색 (Global Search Bar)**:

- 헤더 영역에 항상 노출되는 글로벌 검색 바 (`Ctrl/Cmd + K` 단축키)
- 검색어 입력 시 실시간 자동 완성 (회의 제목, 키워드, 참석자 이름 제안)
- 최근 검색어 히스토리 표시 (최대 10건)
- 디바운스 적용 (300ms) — 타이핑 완료 후 API 호출

**고급 검색 (Advanced Search)**:

- **전문 검색(Full-Text Search)**: 회의 Transcript 본문, 요약, 액션 아이템 내 키워드 검색
- **필터 조합**: 아래 필터를 AND 조건으로 조합 가능
  - 날짜 범위 (시작일 ~ 종료일)
  - 참석자 (다중 선택)
  - 키워드 태그 (다중 선택)
  - 화자 (특정 발언자의 발언만 검색)
  - 회의 상태 (진행 중 / 완료 / 보관)
- **검색 연산자 지원**: `"정확한 구문"`, `+필수단어`, `-제외단어`, `OR` 조합

**검색 결과 표시**:

- **스니펫 하이라이팅**: 검색어 매칭 구간 전후 맥락 텍스트와 함께 하이라이트 표시
- **관련도 점수**: 검색 결과별 관련도 표시 (매칭 횟수, 위치 가중치 기반)
- **정렬 옵션**: 관련도순 (기본), 최신순, 오래된순
- **페이지네이션**: 20건 단위 무한 스크롤 또는 페이지 전환
- **결과 내 재검색**: 검색 결과 범위 내에서 추가 키워드로 좁히기

**검색 → 상세 연동**:

- 검색 결과 클릭 시 해당 회의 상세 페이지로 이동
- Transcript 내 검색어 매칭 위치로 자동 스크롤 및 하이라이트
- 오디오 플레이어를 해당 발언 시점으로 자동 시크(seek)

| US-EDT-03 | 임직원 | 완성된 회의록을 참석자에게 메일로 공유하고 싶다 | 메일 발송 버튼 → 수신자 확인 → 발송 완료 알림 |
| US-EDT-04 | PM | 수정 이력을 확인하고 싶다 | 최종 수정 일시 표시 (향후: 변경 이력 로그) |

#### 4.10.2 상세 요구사항

**수정 기능 (Edit)**:

- **인라인 편집 모드**: 각 섹션(요약, 액션 아이템, 결정사항)에 편집 아이콘 표시, 클릭 시 `contenteditable` 또는 `<textarea>` 전환
- **요약 (TL;DR)**: 멀티라인 텍스트 편집, 마크다운 기본 서식 지원
- **액션 아이템**: 내용(text), 담당자(assignee), 마감일(dueDate), 우선순위(priority) 필드별 개별 수정 가능, 항목 추가/삭제 지원
- **주요 결정사항**: 항목별 텍스트 편집, 항목 추가/삭제 지원
- **키워드 태그**: 태그 추가/삭제 (입력 + Enter로 추가, × 버튼으로 삭제)
- **편집 취소**: 수정 중 취소 버튼 클릭 시 원래 상태로 복원

**DB 저장 기능 (Save)**:

- **저장 버튼**: 편집 모드 활성화 시 상단에 `💾 저장` 버튼 표시
- **저장 API**: `PUT /api/meetings/:id` 호출, 수정된 필드만 전송 (부분 업데이트)
- **저장 피드백**: 성공 시 "저장 완료" 토스트, 실패 시 에러 메시지 표시
- **자동 저장**: 향후 확장 — 편집 후 5초 비활동 시 자동 저장 (디바운스)
- **동시 편집 방지**: 향후 확장 — Optimistic Locking (버전 번호 비교)

**메일 발송 기능 (Email)**:

- **발송 버튼**: 회의 상세 화면 상단에 `📧 메일 발송` 버튼 표시
- **발송 전 확인 모달**:
  - 수신자: 회의 참석자 자동 선택 (체크박스로 개별 해제/추가 가능)
  - 추가 수신자: 이메일 주소 직접 입력 필드
  - 메일 제목: `[회의록] {회의 제목} - {날짜}` 자동 생성 (수정 가능)
  - 미리보기: 발송될 메일 본문 미리보기 영역
- **메일 본문 구성**:
  - 회의 기본 정보 (제목, 날짜, 참석자)
  - AI 요약 (TL;DR)
  - 주요 결정사항
  - 액션 아이템 (담당자, 마감일 포함)
  - 키워드 태그
- **발송 API**: `POST /api/meetings/:id/send-email`
- **발송 이력**: 발송 일시 및 수신자 기록 (향후)

#### 4.11.2 상세 요구사항

**저장 폼 (AI 요약 결과 하단에 표시)**:

- **회의 제목** (필수): 텍스트 입력
- **참석자**: 쉼표 구분 텍스트 → JSON 배열로 변환
- **날짜**: 기본값 오늘 (`new Date().toISOString().slice(0,10)`), date picker
- **시간**: 기본값 현재 시각, time picker
- **소요 시간**: STT meta의 `totalDuration`(초)을 분 단위로 자동 변환, 수동 수정 가능

**자동 매핑 (사용자 입력 불필요)**:

| AI 결과 필드 | → DB 컬럼 |
|-------------|-----------|
| `summaryResult.summary` | `ai_summary` |
| `summaryResult.keyDecisions` | `key_decisions` (JSON) |
| `summaryResult.actionItems` | `action_items` (JSON) |
| `summaryResult.keywords` | `tags` (JSON) |
| `summaryResult.sentiment` | `sentiment` |
| `transcriptResult.segments` | `transcript` (JSON, `[{speaker, time, text}]` 형태) |
| `transcriptResult.fullText` | `full_text` |

**저장 프로세스**:

1. 사용자가 폼 입력 → `💾 DB에 저장하기` 버튼 클릭
2. 프론트엔드: `createMeeting()` API 호출 (`POST /api/meetings`)
3. 백엔드: meetings 테이블에 INSERT, 생성된 `id` 반환
4. 프론트엔드: 성공 토스트 → `router.push('/meetings/${id}')`

**유효성 검증**:

- 제목 미입력 시 저장 차단, 에러 메시지 표시
- AI 요약 미완료 상태에서도 저장 가능 (STT 결과만으로 저장)

## 7. 비기능 요구사항

### 7.1 성능

| 항목 | 목표 |
|------|------|
| STT 처리 시간 | 오디오 1분당 10초 이내 |

### 7.2 보안

| 항목 | 요구사항 |
|------|----------|
| API Key 관리 | 서버 사이드 환경변수 (.env) 관리, 클라이언트 노출 차단 |
| 파일 보안 | 업로드 파일 처리 후 즉시 삭제, 임시 디렉토리 자동 정리 |

| 항목 | 목표 |
|------|------|
| 서비스 가용률 | 99.5% (업무 시간 기준) |
| 에러 처리 | 외부 API 장애 시 graceful degradation |
| 데이터 백업 | 일 1회 자동 백업 (운영 환경) |
