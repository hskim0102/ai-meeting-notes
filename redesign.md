# AI Smart Meeting Notes - 리디자인 제안서

> 작성일: 2026-03-28
> 현재 버전 분석 기반 기능 확장 및 UI/UX 개선 제안

---

## 1. 현재 시스템 요약

| 영역 | 현재 구현 상태 |
|------|---------------|
| 음성 → 텍스트 (STT) | Whisper API, 25MB 초과 시 자동 분할 |
| AI 요약 | Dify Workflow (요약, 핵심결정, 액션아이템, 감정분석) |
| 회의 관리 | CRUD, 상태관리, 태그, 참석자 |
| 액션아이템 | 체크박스 토글, 담당자/기한 표시 |
| 회의실 예약 | 회의실 목록, 주간 캘린더, 예약/취소 |
| 검색 | 키워드 가중치 검색, 날짜/참석자/태그 필터 |
| 이메일 발송 | 수신자 선택 + 미리보기 (SMTP 설정 시 실제 발송) |

## 2. 추가 기능 제안

### 2.1 사용자 인증 및 권한 관리 (Priority: HIGH)

**현재 문제:** 인증 없이 모든 데이터에 누구나 접근 가능

**제안:**
- OAuth 2.0 소셜 로그인 (Google Workspace / Microsoft 365)
- JWT 기반 세션 관리
- 역할 기반 접근 제어 (RBAC)
  - `admin`: 시스템 설정, 회의실 관리
  - `manager`: 팀 회의록 전체 열람, 액션아이템 할당
  - `member`: 본인 참여 회의록만 열람/편집
- 팀/부서 단위 조직 구조 반영

### 2.2 실시간 협업 기능 (Priority: HIGH)

**현재 문제:** 회의 진행 중 실시간 공유 불가, 동시 편집 미지원

**제안:**
- **라이브 트랜스크립션 스트리밍**: WebSocket으로 STT 결과를 실시간 브로드캐스트
- **동시 편집 잠금**: 편집 중인 사용자 표시 + 낙관적 잠금(optimistic locking)
- **실시간 알림**: 새 회의록 생성, 액션아이템 할당, 기한 임박 시 브라우저 푸시 알림
- **코멘트 기능**: 회의록 특정 구간에 스레드형 코멘트 추가

### 2.3 AI 기능 강화 (Priority: HIGH)

**현재 문제:** 단일 요약만 제공, 화자 구분 없음

**제안:**
- **화자 분리 (Speaker Diarization)**: Whisper + pyannote 연동으로 "누가 무슨 말을 했는지" 자동 구분
- **후속 질문 챗봇**: 회의록 기반 Q&A (예: "지난 회의에서 마케팅 예산 관련 결정은?")
- **회의 간 연결 분석**: 동일 프로젝트/주제 회의들을 자동 그룹핑, 의사결정 히스토리 타임라인
- **자동 아젠다 생성**: 이전 회의 미완료 액션아이템 + 반복 주제로 다음 회의 아젠다 초안 생성
- **다국어 지원**: 영어/일본어/중국어 회의 자동 감지 및 번역 요약

### 2.4 액션아이템 고도화 (Priority: MEDIUM)

**현재 문제:** 단순 체크박스만 제공, 알림/추적 부재

**제안:**
- **우선순위 레벨**: 긴급/높음/보통/낮음 (색상 구분)
- **상태 워크플로우**: 대기 → 진행중 → 검토중 → 완료
- **자동 리마인더**: 기한 D-3, D-1, D-day 이메일/슬랙 알림
- **연체 대시보드**: 기한 초과 아이템 자동 에스컬레이션 뷰
- **하위 작업(Sub-task)**: 큰 액션아이템을 세분화
- **Jira/Linear 연동**: 액션아이템 → 이슈 트래커 자동 생성

### 2.5 회의실 예약 고도화 (Priority: MEDIUM)

**현재 문제:** 단순 예약/취소만 가능, 반복 예약 미지원

**제안:**
- **반복 예약**: 매주/격주/매월 정기 회의 등록
- **참석자 자동 추천**: 이전 회의 참석자 기반 추천
- **회의실 자동 배정**: 참석 인원 + 필요 장비 기반 최적 회의실 추천
- **체크인/노쇼 관리**: 예약 후 10분 내 미사용 시 자동 해제
- **Google Calendar / Outlook 연동**: 양방향 캘린더 동기화

### 2.6 리포트 및 분석 (Priority: MEDIUM)

**현재 문제:** 기본 통계(회의 수, 시간)만 제공

**제안:**
- **팀별 회의 분석 대시보드**
  - 주간/월간 회의 시간 트렌드 (차트)
  - 액션아이템 완료율 추이
  - 팀원별 발언 비중 (화자 분리 연동)
  - 감정 분석 트렌드 (회의 분위기 변화)
- **회의 효율성 스코어**: 결정 사항 수 / 회의 시간 비율
- **PDF/Excel 리포트 내보내기**: 주간/월간 회의 요약 보고서 자동 생성
- **비용 분석**: 참석자 수 × 회의 시간으로 추정 비용 산출

### 2.7 외부 연동 (Priority: LOW)

**제안:**
- **Slack/Teams 봇**: 회의 요약 자동 공유, 슬래시 커맨드로 회의 검색
- **Zoom/Google Meet 연동**: 온라인 회의 녹음 자동 가져오기
- **Notion/Confluence 내보내기**: 회의록을 기존 문서 도구로 자동 동기화
- **Webhook**: 회의 생성/완료 시 외부 시스템 트리거

### 2.8 보안 및 감사 (Priority: LOW)

**제안:**
- **감사 로그(Audit Log)**: 누가 언제 어떤 회의록을 열람/편집했는지 기록
- **회의록 잠금**: 확정된 회의록은 편집 불가 처리
- **데이터 보존 정책**: N개월 후 자동 아카이브/삭제 설정
- **내보내기 제한**: 민감한 회의록 다운로드 권한 제어

## 3. UI/UX 개선 제안

### 3.1 다크 모드 지원 (Priority: HIGH)

**현재 문제:** 라이트 모드만 존재, 야간/저조도 환경에서 눈의 피로

**개선안:**
- 시스템 설정 연동 (`prefers-color-scheme`) + 수동 토글 스위치
- Tailwind `dark:` 유틸리티 활용
- 사이드바 상단에 테마 토글 배치
- 다크 모드 컬러 팔레트:
  - 배경: slate-900 → slate-800 (계층 구분)
  - 카드: slate-800, 보더: slate-700
  - 텍스트: slate-100 (본문), slate-400 (보조)
  - 액센트: 기존 primary/accent 색상 밝기 조정

### 3.2 온보딩 및 빈 상태(Empty State) 개선 (Priority: HIGH)

**현재 문제:** 데이터 없을 때 빈 화면만 노출, 첫 사용자 가이드 없음

**개선안:**
- **첫 방문 온보딩 플로우**:
  1. 환영 모달 → 핵심 기능 3단계 투어
  2. 첫 회의록 생성 유도 CTA
- **빈 상태 일러스트레이션**:
  - 대시보드: "아직 회의록이 없습니다" + 일러스트 + "첫 회의 시작하기" 버튼
  - 액션아이템: "할당된 작업이 없습니다" + 완료 축하 일러스트
  - 검색 결과 없음: "검색 결과가 없습니다" + 검색 팁 제안

### 3.3 대시보드 리디자인 (Priority: HIGH)

**현재 문제:** 정보 밀도가 낮고, 핵심 인사이트 부족

- 타임라인형 오늘 일정
- 긴급 액션아이템 우선 노출
- 최근 회의 카드 슬라이더

### 3.4 사이드바 네비게이션 개선 (Priority: MEDIUM)

**현재 문제:** 고정 너비, 모바일 미대응, 접기 불가

**개선안:**
- **접기/펼치기 토글**: 아이콘만 표시되는 미니 모드 (w-16)
- **모바일 대응**: 768px 이하에서 오버레이 드로어로 전환
- **배지 알림**: 미완료 액션아이템 수, 새 회의록 수 배지 표시
- **즐겨찾기/최근 항목**: 사이드바 하단에 최근 본 회의록 3개 퀵 링크
- **키보드 단축키 표시**: 각 메뉴 옆에 단축키 힌트 (예: `⌘+N` 새 회의)

### 3.5 회의 상세 페이지 개선 (Priority: MEDIUM)

**현재 문제:** 탭 전환이 번거롭고, 요약 정보가 분산됨

**개선안:**
- **2컬럼 레이아웃** (데스크탑):
  - 좌측 (60%): 전문(transcript) 뷰어 (타임라인 + 화자별 색상)
  - 우측 (40%): AI 요약 패널 (스크롤 시 sticky)
- **하이라이트 연동**: AI 요약의 핵심 결정 클릭 → 전문에서 해당 구간 하이라이트 스크롤
- **타임스탬프 네비게이션**: 전문 구간 클릭 시 오디오 해당 위치 재생 (오디오 플레이어 내장)
- **인라인 편집**: 탭 전환 없이 요약/액션아이템 직접 편집
- **공유 링크**: 회의록 특정 구간에 대한 딥링크 생성


### 3.6 새 회의 생성 플로우 개선 (Priority: MEDIUM)

**현재 문제:** 단일 페이지에 모든 단계가 나열되어 복잡

**개선안:**
- **스텝 위저드(Step Wizard)** 형태로 단계 분리:
  ```
  ① 녹음/업로드  →  ② STT 변환  →  ③ AI 분석  →  ④ 검토/저장
  ```
- **각 단계별 명확한 진행 상태 표시** (프로그레스 바 + 단계 인디케이터)
- **② STT 변환 단계**: 실시간 변환 진행률 + 예상 소요 시간
- **③ AI 분석 단계**: 분석 중 로딩 애니메이션 + 단계별 진행 (요약 → 결정사항 → 액션아이템)
- **④ 검토 단계**: 전체 결과 미리보기 + 수정 가능 + 저장 전 최종 확인

### 3.7 검색 경험 개선 (Priority: MEDIUM)

**현재 문제:** 별도 페이지에서만 검색 가능, 결과 미리보기 부족

**개선안:**
- **글로벌 커맨드 팔레트 (⌘+K)**:
  - 어디서든 단축키로 즉시 검색
  - 최근 검색어 / 자주 찾는 회의 표시
  - 카테고리별 결과 그룹핑 (회의, 액션아이템, 참석자)
- **검색 결과 인라인 미리보기**: 결과 hover 시 요약 + 핵심결정 팝오버
- **자연어 검색**: "지난달 마케팅 회의에서 예산 관련 결정" 같은 자연어 쿼리
- **저장된 검색**: 자주 사용하는 필터 조합 저장

### 3.8 반응형 디자인 및 모바일 최적화 (Priority: MEDIUM)

**현재 문제:** 데스크탑 중심 레이아웃, 모바일에서 사용 불편

**개선안:**
- **모바일 (< 768px)**:
  - 하단 탭 바 네비게이션 (대시보드/회의/액션/더보기)
  - 카드 목록 단일 컬럼
  - 스와이프 제스처 (좌: 삭제, 우: 완료)
  - 간소화된 회의 상세 (아코디언 형태)
- **태블릿 (768px ~ 1024px)**:
  - 2컬럼 그리드
  - 사이드바 오버레이 모드
- **PWA 지원**: 오프라인 회의록 열람, 홈 화면 추가, 푸시 알림

### 3.9 접근성(A11y) 강화 (Priority: MEDIUM)

**현재 문제:** ARIA 라벨 부재, 키보드 네비게이션 미흡

**개선안:**
- 모든 인터랙티브 요소에 `aria-label`, `role` 속성 추가
- 포커스 인디케이터 강화 (`focus-visible:ring-2`)
- 스크린 리더 호환 (상태 변경 시 `aria-live` 알림)
- 키보드 단축키 체계:
  - `⌘+N`: 새 회의
  - `⌘+K`: 검색
  - `⌘+/`: 단축키 도움말
  - `J/K`: 목록 위/아래 이동
  - `Enter`: 선택, `Esc`: 닫기

### 3.10 로딩 및 전환 경험 개선 (Priority: LOW)

**현재 문제:** 데이터 로딩 시 빈 화면, 페이지 전환 애니메이션 없음

**개선안:**
- **스켈레톤 로딩**: 카드/리스트/차트 영역에 콘텐츠 형태 스켈레톤 표시
- **페이지 전환 애니메이션**: Vue `<Transition>` 활용 페이드/슬라이드
- **낙관적 업데이트(Optimistic UI)**: 액션아이템 토글 시 즉시 UI 반영, 서버 응답 후 보정
- **인피니트 스크롤**: 회의 목록 페이지네이션 대신 스크롤 기반 로딩
- **토스트 알림 개선**: 우측 하단 스택형, 자동 소멸 + 실행취소(Undo) 액션 포함

## 4. 기술적 개선 제안

### 4.1 상태 관리 도입

**현재:** 컴포넌트 로컬 상태 + API 직접 호출
**제안:** Pinia 도입으로 전역 상태 관리
- `useMeetingStore`: 회의 목록, 필터, 캐싱
- `useAuthStore`: 사용자 인증 상태
- `useNotificationStore`: 알림 큐

### 4.2 API 캐싱 전략

**현재:** 매 페이지 진입 시 API 재호출
**제안:**
- Vue Query (TanStack Query) 도입
- 캐시 만료 정책 (staleTime: 30초, gcTime: 5분)
- 백그라운드 리페치 + 낙관적 업데이트

### 4.3 입력 유효성 검증

**현재:** 서버 측 검증 부재
**제안:**
- 프론트엔드: VeeValidate + Zod 스키마
- 백엔드: Zod 미들웨어로 요청 본문 검증
- 공유 스키마로 프론트/백 일관성 확보

### 4.4 테스트 체계

**현재:** 테스트 코드 없음
**제안:**
- 단위 테스트: Vitest (컴포넌트 + 서비스)
- E2E 테스트: Playwright (핵심 플로우)
- API 테스트: Supertest (엔드포인트)

### 4.5 인프라

**현재:** 로컬 개발 환경만 존재
**제안:**
- Docker Compose (MySQL + Express + Vite)
- CI/CD: GitHub Actions (lint → test → build → deploy)
- 에러 트래킹: Sentry 연동
- 모니터링: 요청 로깅 + 성능 메트릭

## 6. 결론

현재 AI Smart Meeting Notes는 핵심 파이프라인(녹음 → STT → AI 요약 → 저장)이 잘 구축되어 있으며, 회의실 예약과 검색 등 부가 기능도 갖추고 있습니다.

**단기적으로는** 다크 모드, 반응형 디자인, 로딩 상태 개선 등 **UX 품질을 끌어올리는 작업**이 사용자 만족도에 가장 큰 영향을 줄 것입니다.

**중기적으로는** 화자 분리, Q&A 챗봇, 대시보드 인사이트 등 **AI 기능 강화**가 경쟁 제품 대비 차별화 포인트가 됩니다.

**장기적으로는** 인증, 실시간 협업, 외부 연동을 통해 **팀 단위 → 조직 단위 확장**이 가능한 엔터프라이즈 제품으로 발전할 수 있습니다.
